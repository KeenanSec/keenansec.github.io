HTB: KEEPER

Keeper is a Linux machine on HackTheBox rated Easy. The attack chain involves default credentials on a ticket management portal, credential reuse for SSH access, and cracking a KeePass database to retrieve the root SSH key.

// RECON

$ nmap -sV -sC -p 22,80 10.10.11.227
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 8.9p1
80/tcp open  http    nginx 1.18.0

Port 80 redirects to tickets.keeper.htb. Added both to /etc/hosts and navigated to the ticket system — it's running Request Tracker (RT).

// INITIAL FOOTHOLD

RT has well-known default credentials: root:password. They worked. Inside the admin panel, user accounts are visible — one user has their initial password stored in the comments field of their profile. 

$ ssh lnorgaard@10.10.11.227
# password from RT user profile
lnorgaard@keeper:~$ cat user.txt

// PRIVILEGE ESCALATION

The home directory contains a zip file with a KeePass .kdbx database and a memory dump. CVE-2023-32784 allows recovering the master password from a KeePass memory dump. 

$ python3 keepass-dump-masterkey.py -d KeePassDumpFull.dmp
Possible password: ●ødgrød med fløde

The password (a Danish dessert name) opens the database. Inside is the root SSH private key stored as a PuTTY .ppk file. Converting with puttygen and SSH-ing as root completes the box. 

$ puttygen root.ppk -O private-openssh -o root.pem
$ chmod 600 root.pem
$ ssh -i root.pem root@10.10.11.227
root@keeper:~# cat root.txt

// TAKEAWAYS

Always check default credentials on admin portals — this is shockingly common in real environments too. The KeePass CVE is a great reminder that memory safety matters even for local password managers.
