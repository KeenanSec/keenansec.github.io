MY CTF TOOLKIT

After dozens of CTFs I've settled on a core set of tools I reach for every time. This isn't an exhaustive list — it's the stuff that actually lives in my workflow.

// RECON

nmap is non-negotiable. I always start with a fast scan then follow up with service detection: 

$ nmap -p- --min-rate 5000 target
$ nmap -sV -sC -p 22,80,443 target

For web recon I use gobuster for directory busting and ffuf for fuzzing subdomains and parameters.

// WEB EXPLOITATION

Burp Suite Community is my proxy of choice. Combined with sqlmap for automated injection testing and manual inspection for logic flaws, it covers most web challenge categories.

// REVERSE ENGINEERING & BINARY

ghidra for decompilation, gdb with pwndbg for dynamic analysis, and pwntools for writing exploits in Python. For simpler RE challenges, strings and file commands catch more than you'd expect.

// CRYPTO

Most crypto challenges come down to recognizing the cipher or vulnerability type. I keep a notes file of common patterns — RSA small exponent, padding oracle, XOR key recovery. CyberChef handles the rest.

// MY SETUP

Kali Linux as my primary CTF VM. I keep a clean snapshot before each competition so I can restore quickly. All tools installed, aliases set, and a ~/ctf/ folder structure ready to go. 

~/ctf/
├── current/     ← active competition
├── archive/     ← past CTFs
└── tools/       ← custom scripts
